## New Feature: Randomized Placeholder Scenarios in Chat Input

### What to add:

In `frontend/index.html`, add a hardcoded array of ~60 example scenarios and display one randomly as placeholder text in the chat input field. When the input is empty and not focused, show the placeholder in a slightly dimmed style. When the user focuses the input, keep the placeholder visible until they start typing (standard HTML placeholder behavior).

### Display behavior:

- On page load and each time the input field is cleared (after sending a message), pick a random scenario from the array using `Math.floor(Math.random() * PLACEHOLDER_SCENARIOS.length)`
- Set it as the input's `placeholder` attribute
- Style the placeholder text with slightly dimmed opacity. Use CSS:
  `input::placeholder { color: var(--color-text-tertiary, #86868b); opacity: 0.6; font-style: italic; }`
- After the user sends a message and the input clears, pick a NEW random scenario (don't show the same one twice in a row — simple do-while loop comparing with the previous value)
- Do NOT use an API call for this. The array is the source, no network requests.
